## Why

Managing test data and troubleshooting application state currently requires direct database access or `iex -S mix` console sessions. An admin panel provides a faster, safer interface for inspecting and manipulating records during development and beyond.

## What Changes

- Add `AshAuthentication` (via Igniter, handled separately as a prerequisite) to introduce user accounts
- Add a `TennisTracker.Accounts` domain with a `User` resource including a `role` enum (`:admin` / `:member`, default `:member`)
- Add `AshAdmin` panel mounted at `/admin`, gated behind authentication and `role == :admin` authorization
- Add `:update` and `:destroy` actions to `TeamType` (currently create-only)
- Add `:destroy` action to `SeasonRules` (currently create/update only)
- Configure all Ash resources for AshAdmin; restrict `TeamMembership` admin to read + destroy only
- Add an admin link to the home page
- Update README to document the admin panel
- Gate all routes (including home page) behind authentication; unauthenticated requests redirect to `/sign-in`
- Add a logout link to the nav bar, visible to authenticated users

## Capabilities

### New Capabilities

- `user-auth`: User resource in Accounts domain with AshAuthentication, role-based access, and dev seeds
- `admin-panel`: AshAdmin panel at `/admin` gated on admin role, covering all Ash resources

### Modified Capabilities

- `team-type`: Add `:update` and `:destroy` actions (currently only create + read)
- `season-rules`: Add `:destroy` action (currently only create + update + read)
- `team-membership`: Constrain admin exposure to read + destroy only
- `home-page`: Add link to admin panel
- `nav`: Add logout link; show only to authenticated users
- `router`: All routes except `/sign-in` require authentication

## Impact

- New dependency: `ash_authentication`, `ash_authentication_phoenix`, `ash_admin`
- New database migration: `users` table, `user_tokens` table (generated by AshAuthentication Igniter)
- New migration: `role` enum column on `users`
- Router gains an `/admin` scope with AshAdmin mount and an authenticated pipeline
- All existing LiveView routes moved inside `ash_authentication_live_session` with `:live_user_required`
- `PageController` home route gains authentication enforcement via plug
- `Layouts.app` gains a `current_user` attr; nav conditionally renders logout link
- `TeamType` and `SeasonRules` gain new actions; no breaking changes to existing callers
- No foreign keys introduced from Tennis resources to Accounts — Tennis remains domain-isolated
